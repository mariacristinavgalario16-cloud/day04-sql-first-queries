#17 — Which new operator (AND, OR, BETWEEN, IN, LIKE) do you find most useful? Why?
Most Useful New Operator
Among all the new operators introduced today, I find LIKE to be the most useful for practical applications.
It allows you to search for patterns within text data even when you do not know the exact value you are looking for, which makes it incredibly flexible in real scenarios.
For example, if you need to retrieve all records where a city name contains a particular word or where a name starts with a specific letter, LIKE handles that with ease using wildcard characters.
This kind of flexible searching is something I can immediately apply in data work, especially when dealing with inconsistent or incomplete entries.
It feels like a smarter and more forgiving way to filter text compared to requiring an exact match.

#19 — How does SQL filtering compare to filtering in Excel? Which feels more natural to you right now?
SQL Filtering vs. Excel Filtering
Comparing SQL filtering to Excel filtering, the most noticeable difference is that Excel is highly visual and interactive,
where you can click through dropdown menus and see results update immediately on screen. SQL, on the other hand, requires you to write out your intent explicitly,
which feels slower at first but ultimately gives you far more precision and control over what data you retrieve. Right now, Excel still feels
more instinctive to me simply because of familiarity, but I recognize that SQL becomes far more powerful when working with large datasets that Excel would struggle to handle efficiently.
The act of writing out filtering logic in SQL also forces you to think more carefully about what you actually want from the data, which I think builds better analytical habits over time.
I look forward to reaching the point where SQL filtering feels just as natural as using Excel.
